업데이트 내역을 공개하는 이유

Keyword Cockpit은 1인 개발로 운영되는 도구이다 보니, 기능이 어떻게 바뀌어 왔는지를 사용자가 직접 확인할 수 있어야 한다고 생각합니다. 이 페이지는 버전이 올라갈 때마다 무엇이 추가되고 무엇이 개선되었는지를 빠짐없이 기록하는 공간입니다. 사소한 변경이라도 사용자 경험에 영향을 줄 수 있는 부분은 모두 남기는 것을 원칙으로 하고 있습니다.

v1.7 — 2026년 7월

  • 노출 레이더 분석 엔진 개선 — 포스팅의 해시태그를 의도 키워드로 우선 사용하도록 변경 (태그가 없으면 제목에서 자동 추출). 검색량 상위 3개 키워드만 선별 스캔해 분석 속도와 정확도를 함께 개선했습니다.
  • 포스팅 지수 산정식 개편 — 노출 순위·최신성·키워드 적합도에 매칭폭 요소를 추가한 4요소 종합 방식으로 개선했습니다.
  • 페이지 구조 개편 — 키워드 분석(/), 블로그 지수(/blog), 노출 레이더(/radar)를 독립 페이지로 분리하고 페이지별 사용법·지표 해설·FAQ를 추가했습니다.
  • 소개(/about), 이용 가이드(/guide), 업데이트 이력(/changelog) 페이지를 신설했습니다.
  • 각 페이지 정보 섹션에 활용 팁·FAQ 보강

v1.6 — 2026년 6월

  • 노출 레이더 신규 출시 — 최근 포스팅 15개의 노출 순위를 자동 스캔해 포스팅 지수를 산출하는 세 번째 도구를 추가했습니다.
  • 키워드 분석에 내 블로그 포스팅 섹션 추가 — 설정에 저장한 블로그 ID 기준으로 검색 키워드 관련 내 글을 자동 표시합니다.
  • 블로그 설정 기능 추가 — 블로그 ID를 저장해 도구 전체에서 공용으로 사용합니다.
  • 서비스 개선을 위한 익명 이용 통계 수집 시작 (30일 보관 후 자동 삭제, IP 미저장).
  • 사이트 파비콘·로고를 속도계 아이콘으로 교체했습니다.

v1.5 — 초기 공개

  • 키워드 분석 공개 — 월간 검색량(PC/모바일), 경쟁도, 연관 키워드, 12개월 검색 트렌드 차트를 제공합니다.
  • 블로그 지수 공개 — 활동 지표(LV.1~9)와 키워드별 상위 100위 내 노출 순위 조회를 제공합니다.
  • 최근 검색 태그, CSV 저장, JSON 복사 등 편의 기능을 제공합니다.

앞으로의 계획

구글 키워드 검색량 연동, 키워드 조합 생성 도구 등을 검토하고 있습니다. 우선순위는 실제 사용 빈도와 사용자 제안을 기준으로 정하고 있으며, 검토 중인 기능이라도 실제 반영 여부와 시점은 유동적일 수 있습니다. 기능 제안이나 오류 신고는 소개 페이지의 문의처로 보내주시면 반영에 참고하겠습니다. 큰 변경 사항은 이 페이지에 버전과 함께 기록되니, 종종 확인해 보시면 도구가 어떤 방향으로 발전하고 있는지 파악하실 수 있습니다.

자주 묻는 질문 (FAQ)

업데이트는 얼마나 자주 이루어지나요?
정해진 주기는 없지만, 사용자 피드백이나 개선이 필요한 부분이 발견되면 비교적 빠르게 반영하는 편입니다. 큰 기능 추가는 한 달에 한 번 정도, 작은 개선 사항은 수시로 이루어집니다.
업데이트 후 이전 검색 기록이나 설정이 사라지나요?
아니요. 블로그 설정 등 사용자 데이터는 브라우저에 저장되는 방식이라 업데이트와 무관하게 유지됩니다. 다만 브라��저 저장 공간 구조가 바뀌는 대규모 업데이트가 있을 경우에는 이 페이지를 통해 사전에 안내드리겠습니다.
베타 기능이나 실험적인 기능도 있나요?
현재는 검증이 끝난 안정적인 기능만 정식 반영하고 있습니다. 새로운 기능을 실험적으로 먼저 공개하게 될 경우, 이 페이지에 베타 표시와 함께 명시하겠습니다.
업데이트 제안이나 오류 신고는 어디로 하나요?
소개 페이지의 문의처로 보내주시면 검토 후 반영합니다. 실제로 지금까지의 업데이트 중 상당수가 사용자 제안에서 시작되었습니다.
과거 버전으로 되돌릴 수 있나요?
별도의 버전 선택 기능은 제공하지 않습니다. 항상 최신 버전이 기본으로 제공되며, 이전 버전에서만 가능했던 동작이 필요하시다면 문의처로 알려주시면 검토하겠습니다.

버전 번호는 어떻게 정해지나요

Keyword Cockpit은 v1.x 형태의 간단한 버전 표기를 사용합니다. 소수점 앞자리가 바뀌는 경우는 노출 레이더처럼 완전히 새로운 도구가 추가되거나 페이지 구조가 크게 바뀌는 시점이고, 소수점 뒷자리는 기존 기능의 개선이나 새로운 지표 추가처럼 상대적으로 작은 변경을 의미합니다. 화면에 드러나지 않는 내부 코드 정리나 사소한 버그 수정은 별도의 버전 번호 없이 조용히 반영되는 경우도 있습니다. 버전 번호의 앞자리와 뒷자리만 봐도 이번 업데이트가 얼마나 큰 변화였는지 대략 가늠하실 수 있도록 이 기준을 유지하고 있습니다.

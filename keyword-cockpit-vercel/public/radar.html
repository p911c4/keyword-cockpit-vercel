노출 레이더 사용법

  1. 블로그 확인 — 스캔 대상 블로그가 상단에 표시됩니다. 다른 블로그를 스캔하려면 변경 버튼(설정)에서 블로그 ID를 저장하세요.
  2. 스캔 시작 — 버튼을 누르면 RSS에서 최근 포스팅 최대 15개를 불러와 순서대로 분석합니다. 진행률 바와 함께 현재 단계가 표시됩니다.
  3. 결과 확인 — 포스팅 5개 단위로 3개 페이지(1~5 / 6~10 / 11~15)에 나뉘어 표시되며, 분석 중인 페이지는 점선 테두리와 모래시계, 완료된 페이지는 체크 표시로 구분됩니다.
  4. 중단·재스캔 — 스캔 중 언제든 중단할 수 있고, 완료 후 다시 스캔 버튼으로 최신 순위를 재확인할 수 있습니다.

포스팅 지수 산정 원리

포스팅 지수는 0.00~1.00 사이의 종합 점수로, 각 포스팅의 검색 노출 상태를 한 눈에 보여줍니다. 포스팅에 입력된 해시태그를 글쓴이의 의도 키워드로 우선 사용하고(태그가 없으면 제목에서 자동 추출), 검색광고 API로 각 키워드의 검색량을 조회해 검색량 상위 3개 키워드만 네이버 블로그 검색 상위 100위 내 노출 여부를 스캔합니다. 아래 네 가지 요소를 종합해 산출하며, 세부 배점 구간은 지속적인 정확도 개선을 위해 내부 기준으로 관리됩니다.

지수가 낮은 포스팅은 제목 앞부분에 핵심 키워드를 배치하고, 검색량이 있는 해시태그로 재정비한 뒤 내용을 보강(리라이팅)하는 것이 효과적인 개선 방향입니다.

자주 묻는 질문 (FAQ)

키워드는 어떻게 정해지나요?
포스팅 작성 시 입력한 해시태그를 의도 키워드로 우선 사용합니다. 해시태그가 없는 포스팅은 제목에서 불용어를 제외한 핵심 단어를 자동 추출합니다. 그중 검색량이 가장 많은 3개만 스캔해 속도와 정확도를 함께 확보합니다.
스캔이 왜 포스팅마다 몇 초씩 걸리나요?
네이버 검색광고 API의 호출 속도 제한을 준수하기 위해 호출 사이에 의도적인 지연을 두고 있습니다. 과도한 요청으로 차단되는 것을 막기 위한 설계입니다.
포스팅 지수는 네이버 공식 지수인가요?
아니요. 공개 API 데이터만 활용한 자체 추정 지표입니다. 공감 수, 방문자 수, 체류 시간처럼 로그인이 필요한 데이터는 반영되지 않습니다.
같은 포스팅인데 스캔할 때마다 지수가 달라집니다.
검색 순위는 시점에 따라 변동되고, 최신성 점수도 시간이 지나며 자연 감소합니다. 주기적으로 재스캔해 추세를 확인하는 방식을 권장합니다.
15개보다 오래된 포스팅도 분석할 수 있나요?
RSS가 제공하는 최근 포스팅 범위 내에서 최대 15개를 분석합니다. 특정 오래된 포스팅의 노출이 궁금하다면 블로그 지수 탭의 키워드 노출 현황 기능으로 개별 확인할 수 있습니다.

리라이팅 우선순위를 정하는 법

스캔 결과에서 지수가 낮은 포스팅이 여러 개라면, 모두 한꺼번에 손보기보다 우선순위를 정하는 것이 효율적입니다. 검색량이 높은 키워드를 다루면서도 지수가 낮은 포스팅을 먼저 개선하면 투입한 노력 대비 효과가 큽니다. 반대로 검색량이 애초에 낮은 키워드를 다룬 글은 지수가 낮더라도 개선 우선순위를 뒤로 미뤄도 괜찮습니다. 키워드 분석 탭에서 해당 키워드의 검색량을 먼저 확인한 뒤 리라이팅 순서를 정하는 것을 추천합니다.

스캔한 결과를 따로 저장할 수 있나요?
현재는 화면에서 결과를 확인하는 방식이며, 별도 저장 기능은 제공하지 않습니다. 스캔은 언제든 다시 실행할 수 있으니 필요할 때마다 재스캔해 최신 상태를 확인하시길 권장합니다.
노출 레이더와 블로그 지수의 차이는 무엇인가요?
블로그 지수는 발행 빈도 등 활동량을 종합한 지표이고, 노출 레이더는 개별 포스팅이 실제 검색 결과에 얼마나 잘 노출되는지를 진단하는 지표입니다. 블로그 전체 흐름은 블로그 지수로, 특정 글의 성과는 노출 레이더로 확인하시면 됩니다.
